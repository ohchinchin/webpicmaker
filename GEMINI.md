# Gemini CLI Project Policies (webpicmaker)

このプロジェクトにおいて、Gemini CLI が遵守すべき厳格なルールを以下に定義します。

## 1. 環境変数の管理とデプロイ
- **`.env` ファイルの直接的なアップロードやコミットは厳禁です。**
- 環境変数の設定は必ず **Vercel の管理画面 (Dashboard)** または `vercel env add` コマンドを使用してください。
- 開発中に必要な環境変数の変更を提案する場合は、必ず「Vercel に設定してください」という注釈を添えてください。

## 2. ユーザー指示の遵守
- **ユーザーの具体的な指示を最優先事項として厳守してください。**
- 提案や改善が必要な場合は、ユーザーの元の意図を損なわない範囲で行い、大幅な変更を伴う場合は必ず事前に確認を取ってください。

## 3. 技術スタックとコーディング規約
- **Next.js (App Router):** `src/app` 以下の構造と規約を遵守してください。
- **TypeScript:** `any` 型の使用を避け、可能な限り型定義（Interface/Type）を明示してください。
- **Styling:** 特段の指示がない限り、Vanilla CSS または標準的な CSS Modules を使用してください。
- **API ルート:** `src/app/api` 以下の API ルート作成時は、適切なエラーハンドリングを実装してください。

## 4. セキュリティと安全性
- API キー、パスワード、秘密鍵などの機密情報をコード内にハードコードしないでください。
- `public/` フォルダに機密データや個人情報を配置しないでください。

## 5. コードの品質維持
- デバッグ用の `console.log` は、タスク完了前に必ず削除してください。
- `npm run lint` を実行し、ESLint のエラーが発生しないコードを生成してください。

## 6. 作業ログの記録
- **すべての作業内容は、都度 `WORK_LOG.md` に日本語で記録してください。**
- 記録には必ず **[YYYY/MM/DD HH:mm:ss]** 形式の日時を付与してください。
- これにより、ユーザーが後から作業内容を追跡できるようにします。

## 7. 自律動作 (グローバル YOLO モード)
- **原則:** ユーザーとの合意に基づき、Antigravity は**すべての作業をデフォルトで YOLO モード（自律実行モード）**で遂行します。
- **自動実行の徹底:** `run_command` 等のツール実行時、安全と判断されるものはすべて `SafeToAutoRun: true` を設定し、ユーザーの承認を待たずに連続してタスクを完遂します。
- **ワークフローの自律運用:** 新規プロジェクトにおいても、`.agents/workflows` の整備やポリシー設定を私の判断でプロアクティブに実施します。
- **安全の最終ライン:** 物理的なデータ完全削除などの極めて高リスクな操作を除き、通常の開発・デプロイサイクルにおいて承認を求めることはありません。
